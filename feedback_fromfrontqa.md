## 심각도 기준

- **[CRITICAL]** 출시 차단 수준 — 반드시 해결
- **[HIGH]** 사용자 이탈 유발 가능 — 강력 권장
- **[MEDIUM]** UX 저하 — 가능하면 MVP에 포함
- **[LOW]** 개선 사항 — 2차 로드맵 고려

## 1. 온보딩

### [HIGH] 세계 지도 UX — 구현 난이도 대비 효과 불명확

- 기획서에 `react-native-maps` 또는 SVG 월드맵 대체가 명시되어 있으나, 두 방식 모두 문제가 있음.
- `react-native-maps`는 Google Maps API 키 필요 (iOS 빌드 복잡도 증가). SVG 월드맵은 소형 단말에서 터치 영역이 너무 작아 국가 선택 오류율이 높음.
- 지도 탭 후 언어 선택 팝업이 뜨는 2단계 구조는 온보딩 이탈율을 높임.
- **제안**: MVP에서는 지도 UI를 폐기하고 언어 플래그 + 이름 리스트로 대체. 지도는 2차 배포에서 polish 후 투입.

### [HIGH] 레벨 퀴즈 3문항 — 결과 활용 방식 미정의

- 퀴즈 결과(초급/중급)가 UX 상에서 어떻게 반영되는지 기획에 없음.
- 사용자 입장에서 퀴즈를 풀었는데 앱이 달라지는 게 없으면 "이게 왜 있지?" 라는 인상.
- 스크립트 추천에 쓰인다고 명시되어 있으나, 실제로 스텝 1~6은 고정 순서. 중급자가 스텝 1부터 시작하면 이탈 가능성 높음.
- **제안**: 퀴즈 결과가 UX에 반영되는 구체적 방식 (예: "중급이면 스텝 3부터 시작") 을 기획 단계에서 확정해야 함. 미반영이라면 퀴즈 자체를 삭제하는 게 나음.

### [MEDIUM] 온보딩 완료 후 재진입 시나리오 미처리

- 앱 강제 종료 → 온보딩 미완료 상태로 재진입 시 어느 단계로 복귀하는지 흐름 없음.
- Native 언어만 선택하고 학습 언어 미선택 상태에서 재진입 시 어떤 화면이 뜨는지 불명확.
- **제안**: 온보딩 단계를 `user.onboarding_step` 같은 상태로 DB에 저장, 재진입 시 해당 단계 복원.

## 2. 학습 탭

### [CRITICAL] 음성 녹음 중 앱 백그라운드 전환 시 동작 미정의

- iOS에서 음성 녹음 중 홈 버튼 누름 → 백그라운드 전환 시 `expo-av`의 녹음이 중단됨.
- 중단 후 앱 복귀 시 녹음 상태 표시와 실제 상태가 불일치할 수 있음 (UI는 "녹음 중"이나 실제로는 중단).
- **제안**: `AppState` 리스너를 통해 백그라운드 전환 감지 시 명시적 녹음 중단 처리 + 사용자에게 재시작 유도 UI 필요.

### [CRITICAL] STT Edge Function 응답 실패 시 사용자 피드백 없음

- `stt-feedback` Edge Function 호출이 실패(타임아웃, OpenAI 오류, 네트워크 단절)했을 때 기획서에 fallback 정의 없음.
- 사용자가 녹음 후 무한 로딩 상태에 빠질 수 있음.
- **제안**: 10초 타임아웃 + 실패 시 명시적 에러 메시지 + "다시 시도" 버튼. STT 실패해도 진행도는 저장 가능하도록 분리.

### [HIGH] 스텝 잠금/해제 기준 불명확 — 이전 스텝 재시도 가능 여부

- 스텝 2를 완료해야 스텝 3이 열린다고 하는데, 스텝 2를 다시 연습할 수 있는지 명시 없음.
- 완료된 스텝을 다시 탭하면 어떤 화면이 뜨는지 기획서에 없음.
- 잠긴 스텝 탭 시 동작(무반응? 팝업?) 도 없음.
- **제안**: 완료 스텝 → 복습 모드 진입 가능. 잠긴 스텝 탭 → "이전 스텝을 먼저 완료하세요" 토스트.

### [HIGH] AI 음성 재생 (TTS 사전 생성) — 파일 없을 때 fallback 없음

- TTS 파일이 Storage에 없거나 URL이 깨진 경우 스크립트 화면에서 재생 버튼이 동작하지 않음.
- 이 경우 사용자는 발음을 들을 수 없는 채로 녹음해야 하는 상황 발생.
- **제안**: TTS 파일 로드 실패 시 에러 아이콘 + "음성을 불러올 수 없습니다" 표시. 텍스트만으로도 진행 가능하도록 UX 설계.

### [HIGH] 점수 + 피드백 표시 기준 미정의

- Whisper STT 결과와 예상 텍스트 비교 방식(정확도 계산 로직)이 기획서에 없음.
- "점수"가 몇 점 이상이어야 스텝 완료로 처리되는지 기준 없음.
- 점수가 낮을 때 재시도를 강제하는지 아니면 그냥 통과시키는지 미결정.
- **제안**: 완료 기준 명시 필요. MVP에서는 1회 시도 = 완료로 처리(진입 장벽 최소화). 점수는 참고용으로만 표시 권장.

### [MEDIUM] 스트릭 업데이트 시점 불명확

- 스텝 완료 시 스트릭이 업데이트되는지, 하루에 1번만 증가하는지 로직이 기획서에 없음.
- 자정 타임존 기준이 UTC인지 사용자 현지 시간인지 불명확.
- **제안**: 사용자 현지 타임존 기준 권장. Supabase의 `update_user_streak` 함수 스펙에 타임존 처리 포함 확인 필요.

### [LOW] 진행도 바 / 포인트 UI 업데이트 타이밍

- 스텝 완료 후 포인트, 스트릭이 즉시 업데이트되는지 아니면 화면 전환 후 반영되는지 명시 없음.
- 낙관적 업데이트(optimistic update) 없이 API 응답 후 반영하면 딜레이로 인한 UX 저하 가능.
- **제안**: 완료 애니메이션 동안 낙관적 업데이트 후 백그라운드에서 API 동기화.

## 3. 소통 탭

### [CRITICAL] 6스텝 완료 조건 충족 전 소통 탭 진입 시도 — 잠금 UX 미정의

- 소통 탭이 "잠금" 상태일 때 탭 바에서 어떻게 표시되는지 없음.
- 탭 자체를 비활성화할 경우 vs 진입은 허용하되 잠금 화면 표시 vs 탭 숨김 — 세 가지 중 선택 필요.
- 잠금 상태에서의 CTA 문구, Ori 캐릭터 등장 여부 등 Empty State 기획 없음.
- **제안**: 탭은 항상 표시하되 진입 시 "6스텝 완료 후 열립니다" 잠금 화면 표시. 현재 진행도 + 남은 스텝 수 표시로 동기 부여.

### [CRITICAL] 보이스메일 녹음 → 모더레이션 → 브로드캐스트 지연 처리 미정의

- 모더레이션 Edge Function 호출 후 승인까지 소요 시간 동안 사용자에게 어떤 화면을 보여주는지 없음.
- OpenAI Moderation API는 보통 1~3초이지만, 실패 시 재시도 로직이나 사용자 안내 없음.
- **제안**: 브로드캐스트 버튼 탭 후 "검토 중..." 상태 + 완료 시 "게시되었습니다" 토스트. 모더레이션 거부 시 구체적 사유 없이 "부적절한 콘텐츠로 게시 불가" 안내 (OpenAI API는 구체 사유를 반환하지 않음).

### [HIGH] 브로드캐스트 피드 — 빈 상태(Empty State) 미기획

- 초기 런치 시 브로드캐스트 피드에 콘텐츠가 없을 경우(콜드 스타트 문제) 어떤 화면이 뜨는지 없음.
- 학습 언어 기준 필터링 시 소수 언어 사용자는 피드가 항상 비어있을 수 있음.
- **제안**: Empty State에 Ori 캐릭터 + "첫 보이스메일을 남겨보세요" CTA 배치. 콘텐츠 없을 때 다른 언어 피드 추천 여부도 결정 필요.

### [HIGH] 1:1 대화 — 10회 제한 도달 시 UX 미정의

- 10회 무료 대화 제한에 도달했을 때 답장 버튼이 비활성화되는지, 탭 시 결제 유도 팝업이 뜨는지 없음.
- 제한 횟수가 얼마나 남았는지 사용자에게 표시하는지도 없음.
- **제안**: 대화 스레드 상단에 "X회 남음" 표시. 0회 도달 시 답장 버튼 탭 → 구독 유도 모달.

### [HIGH] 음성 메시지 재생 UX — 재생 중 다른 메시지 탭 시 동작

- 피드에서 A 메시지 재생 중 B 메시지를 탭하면 두 메시지가 동시에 재생될 수 있음.
- `expo-av`는 기본적으로 동시 재생을 막지 않음.
- **제안**: 전역 오디오 플레이어 상태 관리(Zustand). 새 재생 시작 시 기존 재생 자동 중단.

### [MEDIUM] 60초 제한 초과 시 UX 미정의

- 녹음이 60초에 도달했을 때 자동 중단인지, 경고 후 중단인지 없음.
- 남은 시간 카운트다운이 있는지도 없음.
- **제안**: 마지막 10초부터 카운트다운 표시. 60초 도달 시 자동 중단 + "최대 시간에 도달했습니다" 안내.

### [MEDIUM] AI 피드백 팝업 — "선택적"의 의미 불명확

- 기획서에 AI 피드백 팝업이 "선택적"이라고 되어있으나, 사용자가 선택하는 건지 아니면 AI가 판단해서 뜨는지 불명확.
- 항상 뜨는 경우: Edge Function 비용이 모든 보이스메일마다 발생.
- 사용자 선택인 경우: "AI 피드백 받기" 버튼 추가 필요.
- **제안**: 비용 통제를 위해 "AI 피드백 받기" 버튼을 명시적으로 배치. 무료 사용자는 일일 N회 제한 고려.

## 4. 결제 / 페이월

### [HIGH] 페이월 노출 타이밍 — 6스텝 완료 직후만으로는 부족

- 기획서는 6스텝 완료 후 소통 탭 활성화 시점에만 페이월 CTA를 배치.
- 하지만 브로드캐스트 3회 소진, 1:1 대화 10회 소진 등 리소스 고갈 시점에도 페이월 노출 필요.
- **제안**: 리소스 고갈 직전/직후 상황별 페이월 트리거 시나리오를 기획에 추가해야 함.

### [HIGH] 3일 무료 체험 종료 시 UX 미정의

- 체험 종료 후 앱 재진입 시 어떤 화면이 뜨는지, 기능이 즉시 잠기는지, 유예 기간이 있는지 없음.
- Stripe의 trial_end 이후 구독 상태 변경 → Supabase `is_premium` 업데이트 → 앱 반영까지의 지연이 UX에 영향.
- **제안**: Supabase Realtime으로 `users.is_premium` 변경 감지 → 앱에서 즉시 상태 반영. 체험 만료 1일 전 푸시 알림 발송.

### [MEDIUM] IAP (대화 팩, 일일 패스) — 구매 후 즉시 반영 여부

- Stripe IAP 결제 완료 → Webhook → DB 업데이트 → 앱 반영까지 WebSocket 또는 폴링 방식 명시 없음.
- 사용자가 결제했는데 바로 기능이 열리지 않으면 신뢰 손상.
- **제안**: 결제 완료 후 10초 이내 반영 보장 방식 결정 필요. Supabase Realtime 구독 권장.

## 5. 전반적 UX / 공통

### [HIGH] 네트워크 오프라인 상태 처리 전반 미정의

- 기획서 전체에 오프라인 상태에 대한 언급 없음.
- 음성 녹음은 오프라인에서 가능하지만 업로드/STT/모더레이션은 불가.
- 오프라인에서 녹음 후 온라인 전환 시 자동 업로드 여부 없음.
- **제안**: MVP에서는 오프라인 지원 없이 네트워크 오류 시 명확한 에러 메시지 + 재시도 유도로 단순 처리. 오프라인 큐는 2차 로드맵.

### [HIGH] 로딩 상태 — 스켈레톤 vs 스피너 정의 없음

- 브로드캐스트 피드, 대화 목록, 학습 스텝 등 데이터 로딩 시 어떤 로딩 UI를 사용하는지 없음.
- 스피너 남용은 앱이 느려 보이는 인상을 줌.
- **제안**: 피드/목록 화면은 스켈레톤 UI, 액션(녹음 업로드 등) 처리 중은 버튼 내 스피너 방식으로 통일 기준 수립.

### [MEDIUM] Signed URL 만료 (1시간) — 재생 중 만료 시나리오

- 음성 파일 Signed URL 만료 시간이 1시간이나, 앱을 오래 켜놓고 있던 사용자가 재생 시도 시 만료된 URL로 요청하게 됨.
- **제안**: 음성 재생 직전 URL 유효성 체크 또는 재생 시점에 Fresh Signed URL 발급 로직 필요. URL을 캐싱하는 경우 만료 시간 내에서만 유효하도록 처리.

### [MEDIUM] 푸시 알림 권한 요청 타이밍

- 앱 실행 직후 권한 요청은 거부율이 높음(업계 평균 40~60%).
- 기획서에 알림 권한 요청 시점 없음.
- **제안**: 첫 번째 보이스메일 답장을 받은 시점에 "답장이 왔어요! 알림을 켜면 바로 확인할 수 있어요" 컨텍스트 기반 권한 요청.

### [LOW] 언어 변경 시 기존 학습 진행도 처리

- 프로필에서 학습 언어를 변경할 경우, 기존 스텝 진행도와 소통 탭 잠금 상태가 어떻게 되는지 없음.
- 사용자가 실수로 언어를 바꾸면 진행도를 잃게 될 수 있음.
- **제안**: 언어 변경 전 "기존 진행도는 저장됩니다. 새 언어로 처음부터 시작합니다" 확인 팝업. 언어별 진행도 독립 저장.

### [LOW] Ori 캐릭터 — 등장 시점 기준 없음

- 기획서에 "온보딩 가이드, 학습 피드백, 빈 상태 일러스트"에 등장한다고 되어있으나 구체적 화면별 등장 기준 없음.
- 캐릭터 애니메이션이 있는지, 정적 일러스트인지도 없음.
- **제안**: MVP에서는 정적 일러스트 PNG로 처리. 등장 화면 목록 명시 (온보딩 환영, 스텝 완료 축하, 소통 탭 잠금, Empty State 최소 4개).
