# Oriverse 소비자 심리학 피드백 — 총 결제 금액 극대화

> 기준: 기존 결제 구조(가격, 플랜 종류) 유지. 트리거 시점, 프레이밍, UX 순서만 조정.
> 근거: 실제 앱(Duolingo, Headspace, Spotify, Calm) 사례 + 행동경제학 원칙 기반.

---

## 핵심 원칙 요약

구독 전환율을 높이는 단 하나의 법칙:
**사용자가 "이걸 잃고 싶지 않다"고 느끼는 순간에 결제 화면을 보여줘라.**

지금 기획의 트리거는 대부분 "벽에 막혔을 때"다. 이건 좌절 전환이라 이탈 위험이 있다.
가장 높은 전환율은 "감정적 정점(Emotional Peak)" 에서 나온다.

## 1. 결제 유도 시점 재설계 (가장 중요)

### 현재 트리거 구조의 문제

| 현재 트리거 | 사용자 감정 | 전환 가능성 |
|------------|------------|------------|
| 브로드캐스트 5회 소진 | 좌절, 차단감 | 중간 |
| 1:1 대화 10회 소진 | 차단감 | 중간 |
| 6스텝 완료 직후 | **성취감 최고조** | **높음** |
| 체험 만료 후 재진입 | 상실감 | 중간~낮음 (이미 떠난 사람) |

### 추가해야 할 "감정적 정점" 트리거

**[신규 2] 스트릭 7일 달성 직후**

Duolingo 데이터: 스트릭 7일 이상 사용자의 구독 전환율은 3일 미만 대비 3.5배.
7일은 "습관이 형성됐다"는 심리적 이정표다. 이 시점의 사용자는 이미 투자했다고 느낀다.

- 트리거: 스트릭 7일 달성 축하 화면 → 하단에 "7일 연속 학습자의 78%는 프리미엄으로 목표를 완주했어요" + 구독 CTA
- Ori 캐릭터 등장 화면과 자연스럽게 결합 가능 (추가 개발 최소)

**[신규 3] 브로드캐스트 첫 "좋아요" 또는 답장 수신**

자신의 음성이 전세계 사람에게 닿았다는 경험. 이 순간 사용자는 "더 많이 올리고 싶다"는 욕구를 느낀다. 브로드캐스트 제한(5회/일)은 이 욕구에 맞닿아 있으므로, 제한 메시지 전에 "반응을 더 받고 싶으면" 구독을 연결한다.

- 트리거: 내 브로드캐스트에 첫 답장 or 반응 수신 시 + 오늘 브로드캐스트 3회 이상 사용 시
- CTA: "오늘 이미 X개를 올렸어요. 프리미엄에서 무제한으로."

## 2. 기존 트리거 프레이밍 수정 (손실 회피 적용)

손실 회피(Loss Aversion): 사람은 같은 크기의 이득보다 손실을 2배 강하게 느낀다. — Kahneman & Tversky, 1979

### 지금 CTA vs 수정 CTA

**브로드캐스트 소진 시**

- 현재: "오늘 사용 한도 도달. 구독하면 무제한"
- 수정: "오늘 이미 5개를 올렸어요. 내일까지 기다리거나, 지금 바로 계속하거나."
  - "기다린다"는 선택지를 명시하면 오히려 구독이 능동적 선택처럼 보임 (선택 아키텍처)

**체험 만료 1일 전 푸시 알림**

- 현재: "체험이 내일 끝나요. 지금 구독하면 할인"
- 수정: "이번 주 브로드캐스트 [N]개, 대화 [N]회. 내일이면 이 모든 게 멈춰요."
  - 개인화된 사용 이력 + 손실 프레이밍. Spotify가 연간 리포트(Wrapped)로 이탈을 막는 것과 동일한 원리.
  - Supabase에서 user_daily_stats 조회 후 알림 문구에 삽입하면 됨 (큰 개발 없음)

**1:1 대화 10회 소진**

- 현재: 답장 버튼 탭 → 구독 모달
- 수정: 8회 시점에 스레드 상단 카드로 선제 안내: "이 대화, 2번 더 말할 수 있어요. 계속 이어가고 싶다면?"
  - 0이 되기 전에 불안을 만들어야 함. 0이 되면 이미 포기하는 사람이 생긴다.
  - 현재 "X회 남음" 표시는 있지만, 2회 남았을 때 별도 프롬프트가 없음 → 추가 필요

## 3. 결제 화면 가격 설계 (앵커링 + 디코이 효과)

### 가격 표시 순서 변경

Anchoring: 먼저 보는 숫자가 기준점이 된다. 비싼 것부터 보여줘야 나머지가 싸 보인다.

현재 추정 표시 순서: 월 $4.99 → 연 $39.99 → Lifetime $29

**권장 표시 순서:**

```
[가장 먼저 표시]
연간 플랜 — $39.99/년
"월 $3.33" 로 환산 강조 + "MOST POPULAR" 배지

[두 번째]
월간 플랜 — $4.99/월
"언제든 해지 가능"


### "할인율" 표현 방식

- 현재: "~33% 할인"
- 수정: "2개월 무료" — 같은 수치이지만 무료라는 단어가 전환율을 높임
  - Amazon Prime, Spotify 모두 "2개월 무료"로 연간 플랜 유도
  - $39.99 = "$4.99 × 10개월 + 2개월 무료"로 표현

## 4. 무료 체험 기간 연장 권고 (3일 → 7일)

**근거:**

- Calm 앱: 7일 무료 체험으로 전환율 40% 증가 보고
- Headspace: 2주 체험이 1주 대비 구독 유지율 1.6배
- 핵심 이유: 언어 학습 앱의 가치를 체감하는 데 최소 5~7일이 필요함. 3일은 스텝 2~3까지밖에 못 감 → 소통 탭 경험 불가 → 핵심 가치 미경험 → 전환 불가

**Oriverse 구체적 계산:**

- 3일 체험: 스텝 1~2 완료 → 소통 탭 읽기 전용 미리보기 시작 → 체험 종료. 소통 탭 완전 활성화(6스텝) 경험 불가.
- 7일 체험: 스텝 4~5까지 가능 → 소통 탭 읽기 활성화 → 실제 브로드캐스트 피드 경험 가능 → 가치 체감 후 결제

단, 7일 체험 시 중간에 만료 경고를 2회 이상 보내야 함 (4일째, 6일째).

## 5. 스텝 7 (유료 콘텐츠) 페이월 위치 조정

**현재:** 처음 3개 무료 미리보기 → 4번째에 페이월

**IKEA 효과 (Norton et al., 2011):** 직접 노력을 투입할수록 해당 결과물에 더 큰 가치를 부여한다.
3개는 투입이 너무 적어 미련이 없다.

**권장:** 7개 무료 → 8번째에 페이월

50개 중 7개(14%)는 무료. 절반 이상이 남아있는 상태에서 막힌다 → "이미 시작했는데"라는 완료 욕구(Zeigarnik Effect) 발동.

추가 비용 없음 — DB에서 `is_paid_preview` 기준 숫자만 조정.

## 6. 소비성 IAP 전환율 높이는 방식

**대화 팩($1.99/10회)의 심리적 문제:**

소비성 IAP는 "쓸 때마다 돈을 쓴다"는 인식을 준다. 매 구매마다 의사결정 비용이 발생하고, 이 피로가 누적되면 아예 쓰지 않게 된다 (Decision Fatigue).

**권장: 소비성 IAP를 구독 전환의 다리로 설계**

- 대화 팩 구매 직후 → "대화 팩보다 구독이 더 이득이에요" 비교 카드 표시
  - "대화 팩 $1.99 × 5번 = $9.95/월 vs 구독 $4.99/월. 구독이 2배 저렴해요."
  - 이미 팩을 산 사람은 실제 사용 의향이 검증된 상태. 구독 제안의 적기.

- 일일 브로드캐스트 패스($1.99/1일): 이걸 2회 이상 구매한 사람에게는 "이번 달에 $X 쓰셨어요. 구독이면 $4.99로 매일 무제한이에요" 메시지 자동 발송.
  - Supabase payments 테이블 기반으로 서버 사이드에서 조건 체크 가능.

## 8. 푸시 알림 전략 보완

**현재:** 답장 수신 시 권한 요청 → 체험 만료 1일 전 알림 → 스트릭 유지 알림

**추가 권장 알림 시나리오:**

| 시점 | 문구 방향 | 목적 |
|------|----------|------|
| 스트릭 5일 달성 | "5일 연속 학습 중. 7일 달성하면 특별 배지." | 7일 트리거 포인트로 유인 |
| 스트릭 7일 달성 | "7일 연속! 이 습관을 프리미엄으로 지키세요." | 구독 전환 |
| 브로드캐스트에 첫 외국인 답장 수신 | "[언어] 원어민이 답장했어요." | 즉시 앱 재오픈 유도 → 감정 정점 페이월 |
| 3일간 앱 미사용 (무료 사용자) | "[N]일 스트릭이 위험해요. 지금 돌아오세요." | 이탈 방지 |
| 무료 사용자 D14 (2주 경과) | "이번 달에 [N]개 학습 완료. 스텝 7까지 가면 어디서든 자유롭게 말할 수 있어요." | 성취 요약 → 동기 부여 |

## 9. 구독 화면 사회적 증거 추가

현재 구독 유도 모달에 사회적 증거 요소가 없다. 전환율에 직접 영향을 주는 요소.

**추가할 문구 (하드코딩 가능, 서버 불필요):**

- "지금 [언어] 학습자 [N]명이 프리미엄으로 학습 중" — 런치 초기에는 수동으로 작성
- "구매자 리뷰: '3주 만에 처음 외국인과 대화했어요'" — 베타 테스터 후기 1~2개
- 별점 표시 (App Store 리뷰 연동 또는 초기에는 베타 피드백 기반)

## 10. 무료/유료 한도 설계 재검토

**현재 무료 한도의 심리적 효과 분석:**

| 제한 | 평가 | 이유 |
|------|------|------|
| 브로드캐스트 5회/일 | 적절 | 하루에 5번 올리면 "더 하고 싶다"는 욕구 자연 발생 |
| 1:1 대화 1인당 10회 | 약간 많음 | 10회면 대화가 끝날 때쯤 소진됨 → 이미 관계가 식어있을 수 있음 |
| AI 피드백 5회/일 | 적절 | 5회는 딱 아쉬운 수준 |

**1:1 대화 제한 미세 조정 권고:**

10회보다 **7회**로 줄이는 것을 검토. 7회는 대화가 아직 활발한 중간 지점에서 막히므로, "이 대화 더 하고 싶다"는 욕구가 더 강하다. 다만 UX 민감도가 있으므로 A/B 테스트 필요 — 런치 직후 Mixpanel로 10회 vs 7회 이탈/전환율 비교 권장.
